# Pipe Instrument ‚Äî Claude Code Plan

A camera-tracked, gesture-controlled instrument built from a 20mm PVC pipe, 3D-printed ArUco clips, and a printed template. No strings. No tuning. No expensive hardware.

## Registry ‚Äî Single Source of Truth

`instrument-registry/registry.json` defines every clip ID across the entire instrument series. The pipe instrument reads from this at runtime ‚Äî no tuning config lives inside the app itself.

**Critical rule:** Clip IDs are permanent once printed. Registry entries are never reassigned, only extended. ID 2 always means G Pentatonic, forever.

### Top Clip IDs ‚Äî Tuning

| ID | Name | Root | Divisions | Strings | Mode |
|---|---|---|---|---|---|
| 0 | G Standard | G3 | 24 | 0, 7, 14 | chromatic |
| 1 | D Standard | D3 | 24 | 0, 7, 14 | chromatic |
| 2 | G Pentatonic | G3 | 15 | 0, 7, 14 | pentatonic |
| 3 | D Minor | D3 | 24 | 0, 5, 10 | diatonic |
| 4 | A Beginner | A3 | 12 | 0, 7, 14 | pentatonic |
| 5 | G Microtonal | G3 | 24 | 0, 7, 14 | microtonal |

### Bottom Clip IDs ‚Äî Mode

| ID | Name | Behaviour |
|---|---|---|
| 1 | Standard | Default ‚Äî free glide, normal dynamics |
| 2 | Snap | Quantise pitch to nearest scale degree |
| 3 | Latch | Bow gesture latches tone until next gesture |
| 4 | Loop | Gesture triggers a looped phrase |
| 5 | MIDI Out | Pipe becomes MIDI controller, no internal sound |
| 6 | Duet | Second pipe in frame shares coordinate space |

### No Clip / Missing Marker

Camera sees no bottom marker ‚Üí silently default to mode ID 1 (Standard). Basic play requires only the top clip.

### Control Token IDs (shared across instrument series)

| ID | Name | Action |
|---|---|---|
| 10 | Sustain | sustain_on |
| 11 | Reverb | reverb_on |
| 12 | Record | loop_record |
| 13 | Silence | mute_all |
| 14 | Reset | reset_state |

## Module 1 ‚Äî ArUco Detection + Registry Lookup (`lib/aruco.js` + `hooks/useAruco.js`)

**Goal:** Detect two ArUco markers, establish coordinate space, and resolve each clip ID against the registry to load tuning and mode config live.

### Tasks

- [ ] Integrate `js-aruco2` or WebAssembly ArUco port
- [ ] Detect all visible markers per frame from canvas ImageData
- [ ] Identify top clip and bottom clip by ID range
- [ ] Resolve top clip ID ‚Üí `registry.top_clips[id]` ‚Üí tuning config
- [ ] Resolve bottom clip ID ‚Üí `registry.bottom_clips[id]` ‚Üí mode config
- [ ] Extract centre point of each marker
- [ ] Compute pipe vector: `top ‚Üí bottom`
- [ ] Project any screen point onto that vector ‚Üí normalised 0.0‚Äì1.0
- [ ] Handle tilt: use vector angle, not raw Y
- [ ] If top clip missing ‚Üí show "No tuning clip detected" warning
- [ ] If bottom clip missing ‚Üí silently use mode ID 1
- [ ] Smooth marker positions over N frames to reduce jitter
- [ ] On clip ID change ‚Üí hot-reload config without restarting audio

### Key Output

```js
{
  calibrated: true,
  pipeVector: { angle: 88.3, lengthPx: 412 },
  tuning: { root: 'G3', divisions: 24, strings: [0,7,14], mode: 'chromatic', name: 'G Standard' },
  playMode: { name: 'Standard', mode: 'default' },
  normalise(screenPoint) ‚Üí float 0.0‚Äì1.0
}
```

## Module 2 ‚Äî Pose Tracking (`hooks/usePose.js`)

**Goal:** Track left hand position and right arm angle reliably from front-facing camera.

### Tasks

- [ ] Integrate MediaPipe Hands (two-hand mode) or Holistic
- [ ] Identify left vs right hand from handedness output
- [ ] Left hand: extract wrist / index finger tip position on each frame
- [ ] Right hand/arm: extract wrist + elbow landmarks
- [ ] Compute right arm angle relative to horizontal
  - `atan2(elbow.y - wrist.y, elbow.x - wrist.x)`
  - Three zones: left (-30¬∞ to -10¬∞), centre (-10¬∞ to +10¬∞), right (+10¬∞ to +30¬∞)
  - Configurable thresholds in `config.js`
- [ ] Run detection in a Web Worker to avoid blocking main thread
- [ ] Smooth all landmarks with exponential moving average

### Key Output

```js
{
  leftHand: { x, y, visible },
  rightArm: { angle, zone: 0|1|2, wrist: {x,y}, elbow: {x,y} }
}
```

## Module 3 ‚Äî Gesture Classifier (`lib/gesture.js`)

**Goal:** Convert raw landmark data into musical gestures, respecting the active play mode from the registry.

### Gestures to Detect

| Gesture | Detection Method |
|---|---|
| **Pitch position** | Left hand Y ‚Üí normalised via ArUco ‚Üí note index |
| **String select** | Right arm zone (0/1/2) ‚Üí string 1/2/3 |
| **Bow** | Right wrist horizontal velocity > threshold, sustained > 80ms |
| **Pluck** | Right wrist velocity spike > threshold, duration < 80ms |
| **Vibrato** | Left hand Y oscillation: ¬±N px at > 4Hz |
| **Slide** | Left hand Y changing while contact maintained |
| **Rest** | Right arm stationary, low velocity |

### Tasks

- [ ] Implement velocity calculator (delta position / delta time)
- [ ] Implement oscillation detector (FFT or zero-crossing on Y history buffer)
- [ ] State machine: IDLE ‚Üí BOWING / PLUCKING / RESTING
- [ ] Debounce pluck trigger (ignore re-triggers within 100ms)
- [ ] Bow: track direction (up/down stroke) for dynamics
- [ ] Bow speed ‚Üí map to amplitude/pressure
- [ ] Configurable thresholds per gesture in `config.js`
- [ ] Respect active play mode: snap mode suppresses slide output; latch mode changes state machine

### Key Output

```js
{
  pitch: { noteIndex: 14, hz: 440.0, sliding: true },
  string: 1,          // 0, 1, 2
  gesture: 'bow',     // 'bow' | 'pluck' | 'rest'
  bowSpeed: 0.72,     // 0.0‚Äì1.0
  vibrato: { active: true, rate: 5.2, depth: 0.3 }
}
```

## Module 4 ‚Äî Pitch Mapping (`lib/pitch.js`)

**Goal:** Convert normalised position (0.0‚Äì1.0) and string index to Hz, using only registry-sourced tuning config.

### Tasks

- [ ] `positionToNote(normalisedY, stringIndex, tuningConfig)` ‚Üí MIDI note number
- [ ] `noteToHz(midiNote)` ‚Üí frequency (equal temperament)
- [ ] Glide mode: return continuous Hz when `sliding: true`
- [ ] Snap mode: quantise to nearest scale degree when play mode is `snap`
- [ ] Microtonal mode: subdivide semitones per registry config
- [ ] No hardcoded tuning presets in this module ‚Äî all config from registry

## Module 5 ‚Äî Synthesis Engine (`lib/synth.js`)

**Goal:** Produce realistic bowed and plucked string tones using Tone.js.

**Bowed voice:**
- [ ] Tone.js `FMSynth`: attack 80ms, decay 0, sustain 1.0, release 200ms
- [ ] Bow speed ‚Üí modulation depth (pressure feel)
- [ ] Bow direction change ‚Üí brief amplitude dip (natural bow change)
- [ ] Vibrato: LFO on frequency, rate + depth from gesture data

**Plucked voice:**
- [ ] Karplus-Strong or `PluckSynth` ‚Äî sharp attack, exponential decay
- [ ] Velocity from pluck speed

**String switching:**
- [ ] Three voice instances, one per string
- [ ] Crossfade ~20ms on string switch to avoid clicks

**Pitch glide:**
- [ ] `synth.frequency.rampTo(hz, 0.02)` for smooth slides
- [ ] No ramp in snap mode

**Global:**
- [ ] Boot audio context on first user gesture (browser requirement)
- [ ] `startBow(hz, pressure)`, `updateBow(hz, pressure)`, `stopBow()`
- [ ] `triggerPluck(hz, velocity)`
- [ ] `setVibrato(rate, depth)`
- [ ] Master reverb (Tone.js `Reverb`) + limiter

## Module 7 ‚Äî Hardware Generation (`hardware/generate.js`)

**Goal:** Single command regenerates all clip files whenever the registry changes.

```js
import { generateSet } from 'aruco-3d';
import registry from 'instrument-registry/registry.json';

// Top clips ‚Äî tuning
generateSet({
  registry:    registry.top_clips,
  form:        'pipe-clip',
  pipe_od:     20.4,
  strategy:    'raised',
  label_back:  true,   // emboss clip name label on reverse face
  output_dir:  './hardware/clip/generated/'
});

// Bottom clips ‚Äî mode
generateSet({
  registry:    registry.bottom_clips,
  form:        'pipe-clip',
  pipe_od:     20.4,
  strategy:    'raised',
  label_back:  true,
  output_dir:  './hardware/clip/generated/'
});
```

All `.scad` and `.3mf` files are generated output ‚Äî never edited by hand.
